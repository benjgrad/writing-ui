## ðŸ§© The 5-Dimensional Context Framework

Once a document passes the gatekeeper, it must be "fingerprinted." To ensure your future self can actually find and use this note, every new entry should be able to answer these five contextual dimensions:

### 1. The Project Dimension (The "Where")

* **Question:** What specific output does this move forward?
* **Constraint:** If it doesn't link to a project, it must link to a **"Seedling Area"** (a curiosity you are actively nurturing).

### 2. The Field Dimension (The "What")

* **Question:** In what domain of knowledge does this live?
* **Constraint:** Use broad tags (e.g., `#CognitivePsychology`) rather than hyper-specific ones to allow for serendipitous discovery later.

### 3. The Intent Dimension (The "Why")

* **Question:** Am I trying to *learn* this, *refute* this, or *use* this as a tool?
* **Constraint:** This defines the "tone" of the note. A "Refutation" note looks very different from a "Manual" note.

### 4. The Entity Dimension (The "Who/Which")

* **Question:** What are the "nouns" involved? (People, Software, Books, Laws).
* **Constraint:** These act as the hooks. Linking to a specific **Entity** (e.g., `[[Marcus Aurelius]]`) allows you to see every thought you've ever had regarding that person across different projects.

### 5. The Temporal/Evolutionary Dimension (The "When")

* **Question:** Is this a "Permanent Note" (final thought) or a "Fleeting Note" (work in progress)?
* **Constraint:** Every note should have a status property: `Status: Seed / Sapling / Evergreen`.

* **The "Double-Entry" Rule:** If you find yourself writing the same realization twice, **Merge** them immediately. Do not create a second document.
* **The "Dead-End" Rule:** If a note has been in your system for 6 months and has **zero** outgoing or incoming links, it is "Digital Noise." Either link it or delete it.
